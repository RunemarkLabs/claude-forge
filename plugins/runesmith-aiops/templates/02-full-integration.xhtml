{COMPANY} - Full Integration Guide

End-to-end setup for the AIOPS skills against {COMPANY}'s Atlassian site at {SITE}.

1. Generate credentials

  1. API token: id.atlassian.com/manage-profile/security/api-tokens
  2. Note the email address tied to the token (must be the user's account email).
  3. GitHub PAT (for plugin sync): github.com/settings/tokens - needs repo scope.

2. Configure local .credentials

bash ATLASSIAN_API_TOKEN= GITHUB_PAT= # Optional defaults ATLASSIAN_CONFLUENCE_SPACE_ID={SPACE_ID} ATLASSIAN_JIRA_PROJECT_KEY={PROJECT_KEY} ]]>

3. Verify connectivity

  1. Run /core:setup - performs health checks against Confluence, Jira, GitHub.
  2. Run /atlassian:project-status - read-only Jira call to confirm project access.

4. First write

  1. Run /atlassian:session-log - creates a Confluence page from a draft.
  2. Confirm the published page has full storage formatting (headings, code, tables).

5. Common gotchas

  • Email mismatch with token → 401. Use the email of the account that minted the token.
  • Wrong space id → page lands in wrong space. Always pass space-id={SPACE_ID} on lookups.
  • Missing version bump on update → 409. Skills handle this automatically; manual API users must GET first.
