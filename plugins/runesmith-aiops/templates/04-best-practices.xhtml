{COMPANY} - AIOPS Best Practices

Consent before writes

Every Jira ticket and Confluence page require an explicit user trigger phrase ("make the ticket", "publish the page"). See consent.md. Skills hold drafts in /drafts/ until the trigger fires.

Use canonical tokens

Always use {COMPANY}, {SITE}, {SPACE_KEY}, {SPACE_ID}, {PROJECT_KEY}. No synonyms ({ORG}, {TEAM}, {TENANT}).

Always scope page lookups by space

Title-only searches return cross-space hits. Always include space-id={SPACE_ID}.

Bump version on PUT

GET the page first, read version.number, PUT with number+1. Skills handle this automatically; manual API users must do it explicitly.

Prefer /search/jql POST

The GET /rest/api/3/search endpoint is deprecated on Cloud. Use POST /rest/api/3/search/jql.

Resolve users by accountId

Jira Cloud rejects assignee.name. Look up email → accountId via GET /rest/api/3/user/search.

Severity is custom

Severity is not a default Jira field. Map to priority, or set ATLASSIAN_BUG_SEVERITY_FIELD in .credentials to your custom field id.

Keep .credentials out of git

The marketplace ships .gitignore entries and a .credentials.example. Never commit real credentials. Never echo full credential values in chat.
