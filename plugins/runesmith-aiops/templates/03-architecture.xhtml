Auth flow

  1. Skill resolves .credentials via credentials.md rules.
  2. Builds Basic auth header: email:token base64.
  3. Issues request to {ATLASSIAN_API_URL} with the header.
  4. On 401: re-read once, then prompt user to re-run /core:setup.

Data flow (page create)

  1. User invokes write skill (e.g. /atlassian:feature-doc).
  2. Skill drafts markdown locally to /drafts/....
  3. User reviews and issues consent trigger phrase.
  4. Skill converts markdown → Confluence storage XHTML per confluence-format.md.
  5. Skill POST /wiki/api/v2/pages against {ATLASSIAN_API_URL}.
  6. Skill reports page URL.
