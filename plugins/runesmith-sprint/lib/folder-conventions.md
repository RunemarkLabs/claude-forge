# Workspace Folder Conventions

Single source of truth for the canonical workspace layout. Every workspace-side skill in this marketplace reads from this file when classifying or routing files. Plugins outside `runesmith-workspace` keep their own copy of this lib (each plugin is self-contained).

## Principle

The root stays clean. Only marketplace-standard files, workspace config, and canonical directories live there. Anything else at root is treated as an inbox-item: classify, route, archive.

`_INBOX/` is a permanent feature of the workspace. Its contents are transient — the user drops files into it, `runesmith-workspace:inbox` reads, classifies, and routes them to their canonical homes. After every inbox run, `_INBOX/` ends empty (except for items the user explicitly opted to leave).

## Canonical directories

```
<workspace-root>/
├── _INBOX/                              user drop zone. inbox skill empties.
│
├── plans/
│   ├── active/<slug>/
│   │   ├── plan.md                      required. see lib/plan-format.md
│   │   ├── decisions.md                 optional, append-only decision log
│   │   ├── refs/                        research + source refs scoped to plan
│   │   └── tickets/<KEY>.json           pre-push Jira ticket JSON drafts
│   └── archive/<YYYY-MM>/<slug>/        completed plans (carries refs+tickets)
│
├── notes/                               session notes, handoffs.
│                                        flat. <YYYY-MM-DD>-<slug>.md naming.
│
├── drafts/                              pre-publish Confluence content.
│   ├── features/<slug>/                 feature pages: slug.md + assets/
│   ├── project-docs/<slug>/             project docs
│   └── bugs/<slug>/                     bug docs
│
├── research/<topic>/                    standalone analysis not tied to a plan.
│                                        topic-folder with topic.md + assets.
│                                        Migrates to plans/active/<slug>/refs/
│                                        once a plan adopts it.
│
├── source-docs/<topic>/                 external uploads being processed.
│                                        Vendor PDFs, exported docs, transcripts.
│                                        Folder per topic; raw files inside.
│
├── archive/
│   ├── _pre-migration/<ISO>/            reallocate snapshots
│   ├── _pre-cc-bootstrap/<ISO>/         bootstrap-cc snapshots
│   ├── _pre-atlassian-enable/<ISO>/     atlassian:enable snapshots
│   ├── superseded/<YYYY-MM>/            consumed content, dead docs
│   └── tickets-pushed/<YYYY-MM>/<KEY>.json  ticket JSON after Jira push (history)
│
├── {PROJECT}.cc/                        CC head per lib/cc-workspace.md
│
└── plugins/, dist/, scripts/, .github/  marketplace dev artifacts (only present
                                         in this marketplace's own workspace).
```

## Root keep-list

These are the only top-level entries permitted at workspace root. Anything else gets routed via the inbox classification protocol.

**Marketplace standard files** (only in marketplace dev workspaces):
- `README.md`, `CHANGELOG.md`, `CONTRIBUTING.md`, `INSTALL.md`
- `LICENSE`, `NOTICE`
- `RELEASE.md`, `PUBLIC.manifest.txt` (private to the marketplace dev workspace)

**Workspace config**:
- `CLAUDE.md` — workspace constitution (carries the folder-conventions marker section)
- `STRUCTURE.md` — generated by reallocate from `lib/STRUCTURE.template.md`
- `.credentials`, `.credentials.example`
- `.gitignore`, `.gitattributes`

**Required dirs**:
- `_INBOX/`, `plans/`, `notes/`, `drafts/`, `research/`, `source-docs/`, `archive/`
- `{PROJECT}.cc/` if a CC head exists
- `.claude-plugin/`, `.github/` (marketplace dev only)
- `plugins/`, `dist/`, `scripts/` (marketplace dev only)

Anything else at root = inbox-item.

## Destination map (for inbox classification)

| Inbox content | Destination |
|---|---|
| handoff (`<YYYY-MM-DD>-handoff-<slug>.md` or content matches handoff template) | `notes/<YYYY-MM-DD>-handoff-<slug>.md` |
| session note (loose markdown) | `notes/<YYYY-MM-DD>-<slug>.md` (add date prefix if missing) |
| plan proposal (frontmatter has `slug:` + `status:` + Problem/Decision sections) | `plans/active/<slug>/plan.md` |
| decision record (filename `decisions*.md` or `## YYYY-MM-DD —` headers) | `plans/active/<slug>/decisions.md` (append) — ask which plan |
| Jira ticket JSON draft | `plans/active/<slug>/tickets/<KEY>.json` — ask which plan |
| Confluence draft (feature/bug/architecture template structure) | `drafts/{features,bugs,project-docs}/<slug>/` |
| research / analysis doc | `plans/active/<slug>/refs/` if plan-bound; else `research/<topic>/` |
| external upload (PDF, DOCX, XLSX, transcript, screenshot) | `source-docs/<topic>/` — ask topic |
| screenshot already consumed for content | `archive/superseded/<YYYY-MM>/` |
| misplaced comm (frontmatter `from:` + `to:` + `type:`) | `{PROJECT}.cc/comms/open/<filename>` — warn user comms shouldn't arrive via inbox |
| superseded / "don't look again" | `archive/superseded/<YYYY-MM>/` |
| unclassifiable | leave in `_INBOX/`, surface to user with file head |

## Lifecycle rules

**Tickets**:
1. Drafted in `plans/active/<slug>/tickets/<KEY>.json` by `runesmith-sprint:plan-to-tickets`
2. Pushed to Jira; plan's frontmatter `tickets:` array gains the key
3. Pushed JSON moves to `archive/tickets-pushed/<YYYY-MM>/<KEY>.json` (referenced only on demand for history)

**Plans**:
1. Created in `plans/active/<slug>/` by `runesmith-core:plan`
2. `status: done` → `runesmith-workspace:reallocate` moves to `plans/archive/<YYYY-MM>/<slug>/` (carries `refs/` and `tickets/`)

**Research**:
1. Lands in `research/<topic>/` when no plan exists yet
2. When a plan adopts the topic, the research migrates to `plans/active/<slug>/refs/`
3. Stale research → `archive/superseded/<YYYY-MM>/research-<topic>/`

**Source-docs**:
1. Lands in `source-docs/<topic>/` when external content is being consumed
2. After content is extracted/quoted/referenced, the raw files go to `archive/superseded/<YYYY-MM>/source-<topic>/`

**Drafts**:
1. Drafted in `drafts/{features,bugs,project-docs}/<slug>/`
2. After publish to Confluence, the draft folder moves to `archive/superseded/<YYYY-MM>/draft-<slug>/`

## Skills that read this file

- `runesmith-workspace:reallocate` — uses the keep-list and destination map to plan migrations
- `runesmith-workspace:inbox` — uses the destination map to classify and route inbox content
- `runesmith-workspace:plan-format` (this plugin's lib) — directory shape under `plans/active/<slug>/`
- `runesmith-sprint:plan-to-tickets` — writes drafts to `plans/active/<slug>/tickets/`, archives post-push to `archive/tickets-pushed/`
- `runesmith-devtools:tech-debt` — scans for stale content in canonical homes

Plugins outside `runesmith-workspace` that need this map keep their own copy as a self-contained `lib/folder-conventions.md`. Keep duplicates in sync when the canonical map changes.
